# v1.29.0

Rilis ini berfokus pada penyempurnaan workflow SPJ dan receiving terencana, penguatan kontrol aksi distribusi dan LPLPO lintas role, perbaikan pelaporan stok Puskesmas, serta peningkatan auditability dan tooling verifikasi lokal.

## Added

- Puskesmas operators now have a read-only stock self-check page that compares the latest usable LPLPO digital stock with the same document's recorded physical stock.
- Receiving Admin now exposes a CSV template download next to the CSV import flow so operators can prepare import files with the expected structure.
- Item master data now supports an optional unique `barcode`, preserved across legacy import paths for future scanner workflows.
- `django-auditlog` object history was added for selected critical models, and bulk user status changes now produce per-row audit entries.
- Local Playwright tooling was expanded to support all six operational app roles for browser-based regression and manual multi-role verification.

## Changed

- Procurement/SPJ workflows were tightened: amendments are now contract-scoped more clearly, amendment numbers are derived from the parent SPJ with reserved suffix space, supplier and funding-source quick-create flows are available in SPJ forms, and `GUDANG` can operate but cannot approve SPJ documents.
- SPJ-linked planned receiving leftovers now have to be corrected through procurement amendments instead of the receiving-side close-items action, keeping procurement as the contractual source of truth.
- Standalone distribution fulfillment now follows the same assignee/fallback pattern as preparation, and generated LPLPO return-to-Puskesmas actions are aligned more strictly with the intended role and staff assignment rules.
- Puskesmas persediaan reporting now aligns more consistently around LPLPO baseline stock plus later confirmed receipts and detailed consumption, with improved facility aggregation and more efficient query behavior.
- Navigation and access rules were cleaned up: `Distribusi` is now presented as `Riwayat Distribusi`, auditor navigation is report-focused, admin history placeholders were removed, and `/settings/` is explicitly limited to superusers plus `ADMIN` and `KEPALA`.

## Fixed

- Procurement forms and numbering now behave more reliably, including restored contract/amendment line controls, safer manual SPJ numbering, and clearer amendment context.
- Receiving admin CSV import was hardened, and planned receiving detail handling was improved for SPJ-linked leftovers.
- Generated LPLPO return actions, distribution queue ordering, and created-date presentation were corrected to match the actual workflow intent.
- Gudang users with the correct expired operate access can now finalize already verified disposal documents without reopening approval responsibilities.
- Puskesmas persediaan reporting now handles all-facility aggregation, zero-value category ordering, and LPLPO eager-loading more consistently.

## Security

- Role-driven navigation and action visibility were tightened further so users only see the workflow actions that match their real authorization paths.
- Playwright/session handling was refreshed to avoid stale Puskesmas auth and CSRF state during local multi-role verification.
- Audit coverage for critical object mutations was strengthened through `django-auditlog` plus row-by-row save paths for sensitive bulk user status changes.

## Platform Notes

- This release continues on Django `6.0.7`.
- Documentation was expanded for auditlog usage, six-role Playwright setup, SPJ/procurement workflow rules, and the new Puskesmas stock self-check behavior.
