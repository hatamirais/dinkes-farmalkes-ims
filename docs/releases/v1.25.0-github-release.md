# v1.25.0

Rilis ini berfokus pada penguatan kontrol akses lintas fasilitas, penyempurnaan alur LPLPO dan distribusi, perluasan pelaporan Puskesmas, serta hardening keamanan pada ekspor, upload, dan akses dokumen.

## Added

- Instalasi Farmasi now has a dedicated manual LPLPO distribution create flow for rollout and catch-up work without forcing immediate backfill of historical Puskesmas documents.
- Puskesmas reporting and request-management surfaces were expanded, including operational reports for penerimaan, pemakaian, persediaan, and rekap persediaan.
- LPLPO reporting now includes an asset-valuation dimension derived from `harga_satuan`, improving stock-value visibility in Puskesmas reporting.
- Sensitive authenticated POST endpoints now have configurable rate limiting with centralized `429` handling.

## Changed

- Facility-scoped access is now enforced more strictly for all non-superusers on affected `puskesmas` and `lplpo` operational, review, print, prefill, and reporting surfaces. A linked facility is now mandatory for those users, while cross-facility access remains limited to superusers.
- LPLPO and distribution workflow coordination was tightened: generated LPLPO distributions keep source-driven quantity locks where intended, LPLPO review handoff is cleaner, and manual LPLPO distribution creation is now available for operational fallback.
- Distribution and outbound reporting screens now provide clearer filters, tab behavior, and print output readability.
- Dashboard movement visuals were refreshed and transfer-metric calculations were cleaned up to reduce noise in stock movement summaries.

## Fixed

- Closed authorization gaps affecting report/export access and several facility-scoped operational surfaces.
- Receiving document attachments are now stored behind authenticated private access rather than exposed as public media.
- CSV and XLSX exports now neutralize formula-prefixed values to reduce spreadsheet formula injection risk.
- Non-finite decimal inputs are rejected earlier before business-rule comparisons and stock-related calculations.
- Addressed transfer-metric inaccuracies, distribution form regressions, and related workflow edge cases in LPLPO-driven distributions.

## Security

- Hardened upload validation and audit logging for file-handling paths.
- Tightened report export authorization and object-level `403` handling on scoped modules.
- Applied follow-up DRY/CodeQL cleanup, including ADMIN-role exemption fixes where needed.

## Platform Notes

- Django was updated to `6.0.6`.
- `backend/requirements.txt` and supporting dependency documentation were normalized for the current stack.
- Test coverage was expanded significantly across `core`, `distribution`, `lplpo`, `puskesmas`, `receiving`, `reports`, and `users`.
